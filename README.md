# 1. 📌 專案前情提要 — 為什麼會有這個 Side Project？

台灣的傳統市場產業非常發達，尤其是政府規劃的菜市場，擁有攤位土地的居民會將攤位出租給攤販，形成一種非常特別、歷史悠久的市場生態。

在這樣的體系之下，衍生出一種常見商業模式：

## ✔ 包租代管（攤位管理商）
- 與多位地主簽約租下大量攤位  
- 再將攤位分配給不同攤販  
- 攤位之間常涉及：
  - 借用
  - 調用
  - 轉租
  - 不同市場間的攤位交換  
- 每月需與多位地主與攤販進行對帳、結帳  
- 人工處理非常容易出錯，帳務量巨大  

在深入了解這個產業後發現：  
**市場攤位的帳務流動，比一般租屋市場複雜好幾倍。**

為了降低出錯率、提高效率，我開發了這套 **Python Desktop Application**，協助攤位管理商進行：

- 攤位紀錄管理  
- 租金自動計算  
- 借用/調用紀錄追蹤  
- 月結帳報表產生  

---

# 2. 🎯 專案目的

1. 降低繁雜帳務的人為錯誤  
2. 建立集中式攤位資料庫  
3. 自動化計算應收/應付  
4. 將攤位歷史紀錄統一管理  
5. 讓沒有技術背景的攤位管理者也能用

---

# 3. 🏗 系統架構概述

> 需要我可以另外補 Mermaid 架構圖

### **Frontend / GUI**
- 使用 Python GUI（Tkinter 或 PyQt）
- 提供直覺操作按鈕、表格與彈窗
- 面向非技術人員

### **Backend**
- 核心邏輯以 Python 撰寫  
- 負責計算租金、處理借調紀錄、產生報表

### **資料儲存**
- 使用 SQLite、CSV 或 JSON（依專案版本調整）
- 儲存：
  - 攤位資訊  
  - 地主/承租人資訊  
  - 借用/調用紀錄  
  - 月結資料

# 4. 🔧 系統功能介紹

## ✔ 攤位管理
- 新增、修改、刪除攤位  
- 記錄攤位詳細資訊（市場、編號、面積、地主、租金…）  
- 查詢是否空置、目前承租人  

---

## ✔ 借用 & 調用紀錄管理
- 記錄攤位借出或借入  
- 管理不同攤位管理商之間的調用  
- 預防遺漏與糾紛

---

## ✔ 月結帳自動化
自動生成該月應收/應付金額，包括：

- 地主 → 管理商（攤位出租收入）  
- 管理商 → 攤販（攤位租金）  
- 不同攤位管理商之間的借調費用  
- 每月總金額合計  

可輸出 CSV / Excel 報表。

---

## ✔ 搜尋與過濾功能
- 依市場查詢  
- 依攤位編號過濾  
- 依承租人查詢  
- 查詢特定時間範圍內的借調紀錄  
- 快速定位問題攤位  

---

## ✔ 易用的 GUI 介面
- 清楚、易懂的按鈕配置  
- 表格呈現資料  
- 彈窗輸入框讓非工程背景使用者也能快速上手

# 6. 心得與開發動機

我觀察到許多傳統市場攤位的管理者仍然依賴：
- 紙本帳簿  
- 分散的 Excel  
- Line 對話記錄  
- 人工計算每月結帳  

這些方法對「多市場、多攤位、多地主、多攤商」的情況來說，非常容易出錯，也很難追蹤歷史紀錄。

在接觸到這個產業後，我深刻體會到資訊化工具對傳統行業的重要性，於是開始開發這個 side project，希望：

- 讓帳務管理更乾淨、更透明  
- 減少人工對帳錯誤  
- 把攤位借調記錄完整保存  
- 幫助代管商人、攤販、地主三方都能清楚掌握自己的數據  

希望這個工具能成為傳統市場邁向數位轉型的一小步。
